---
title: "Split de Pagamentos"
description: "Distribua automaticamente o valor de uma cobrança entre múltiplos recebedores no momento da liquidação."
---
## Como funciona
Ao gerar um QR Code, inclua o array `split` no payload. Quando a transação for paga, o valor líquido é distribuído automaticamente entre os recebedores definidos — sem nenhuma ação adicional da sua parte.
<Info>
  O split incide sobre o **valor líquido** (após a taxa da plataforma). O percentual restante fica automaticamente com o dono da cobrança.
</Info>
## Campos do split
| Campo | Tipo | Descrição |
|---|---|---|
| `username` | `string` | Username da conta StylePay que receberá a fatia. |
| `percentageSplit` | `string` | Percentual a ser repassado. Aceita decimais (ex: `"10.5"`). |
## Regras
<Warning>
  - Percentual **mínimo** por destinatário: `0.1%`
  - Soma total de todos os splits: **máximo `95%`**
  - O mesmo `username` **não pode** aparecer duas vezes no array
  - Todos os destinatários precisam ter conta **ativa** na StylePay
  - O dono da cobrança **não pode** ser destinatário explícito do split
</Warning>
## Exemplo de payload
Inclua o objeto `split` junto ao payload normal de criação de QR Code:
```json
{
  "amount": 100,
  "external_id": "pedido-2025",
  "payer": {
    "name": "Monkey D. Luffy",
    "document": "",
    "email": ""
  },
  "postbackUrl": "",
  "split": [
    {
      "username": "parceiro1",
      "percentageSplit": "20"
    },
    {
      "username": "parceiro2",
      "percentageSplit": "10"
    }
  ]
} 
```

Neste exemplo, sobre um valor líquido de R$ 90,00 (após taxa de 10%):

Recebedor	Percentual	Valor
parceiro1	20%	R$ 18,00
parceiro2	10%	R$ 9,00
Dono da cobrança	70% restante	R$ 63,00
Resposta
Quando o split é aceito, a resposta inclui um resumo detalhado de como o valor será distribuído:

```json
{
  "statusCode": 200,
  "message": "QR Code gerado com sucesso",
  "data": {
    "payment_id": "1e6c9f53-fddb-4994-bc02-fd6fe61007fa",
    "qrcode": "00020101021226870014br.gov.bcb.pix...",
    "qrcode_image": "iVBORw0KGgoAAAANS...",
    "status": "PENDING",
    "split": {
      "total_percentage": 30,
      "split_amount_cents": 2700,
      "owner_amount_cents": 6300,
      "recipients": [
        { "username": "parceiro1", "percentage": 20, "amount_cents": 1800 },
        { "username": "parceiro2", "percentage": 10, "amount_cents": 900 }
      ]
    }
  }
}
Os valores em `amount_cents` consideram arredondamento pelo método **Largest Remainder**, garantindo que nenhum centavo seja perdido na distribuição entre os recebedores. 
```
